Factory(IServiceCollection, Type, Func<IInjector, Type, Object>, Lifetime)

Registers a new service factory with the given type. Factories are also services except that the instantiating process is delegated to the caller. Useful if a service has more than one constructor.

Declaration
public static IModifiedServiceCollection Factory(this IServiceCollection self, Type iface, Func<IInjector, Type, object> factory, Lifetime lifetime)
Parameters
Type Name Description
IServiceCollection self

The target IServiceCollection.
Type iface

The service interface to be registered. It can not be null and can be registered only once.
Func<IInjector, Type, Object> factory

The factory function that is responsible for the instantiation. Its call count depends on the value of the lifetime parameter. Note that the second parameter of the factory is never generic, even if you registered the factory for an open generic interface.
Lifetime lifetime

The Lifetime of the service.
Returns
Type Description
IModifiedServiceCollection
Remarks

You can register generic services (where the iface parameter is an open generic type).

Service(IServiceCollection, Type, String, Type, Lifetime)

Registers a new service with the given implementation.

Declaration
public static IModifiedServiceCollection Service(this IServiceCollection self, Type iface, string name, Type implementation, Lifetime lifetime)
Parameters
Type Name Description
IServiceCollection self

The target IServiceCollection.
Type iface

The service interface to be registered. It can not be null and can be registered only once (with the given name).
String name

The (optional) name of the service.
Type implementation

The service implementation to be registered. It can not be null and must implement the iface interface. Additionally it must have only null or one constructor (that may request another dependecies). In case of multiple constructors you can use the Factory(IServiceCollection, Type, Func<IInjector, Type, Object>, Lifetime) method or the ServiceActivatorAttribute.
Lifetime lifetime

The Lifetime of the service.
Returns
Type Description
IModifiedServiceCollection
Remarks

You may register generic services (where both the interface and the implementation are open generic types). The system will specialize the implementation if you request the concrete service.

WithProxy(IModifiedServiceCollection, Func<IInjector, Type, Object, Object>)

Hooks into the instantiating process of the last registered service. Useful when you want to add additional functionality (e.g. parameter validation).

Declaration
public static IModifiedServiceCollection WithProxy(this IModifiedServiceCollection self, Func<IInjector, Type, object, object> decorator)
Parameters
Type Name Description
IModifiedServiceCollection self

The target IServiceCollection.
Func<IInjector, Type, Object, Object> decorator

The decorator funtion. It must return the decorated instance. The original instance can be accessed via the 3rd parameter of the decorator function.
Returns
Type Description
IModifiedServiceCollection
Remarks

You can't create proxies against generic or instance entries. A service can be decorated multiple times.

Exceptions
Type Condition
InvalidOperationException

When proxying not allowed (see above).
