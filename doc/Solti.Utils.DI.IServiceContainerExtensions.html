Lazy(IServiceContainer, Type, ITypeResolver, Lifetime)

Registers a service where the implementation will be resolved on the first request. It is useful when the implementation is unknown in compile time or you just want to load the containing assembly on the first request.

Declaration
public static IServiceContainer Lazy(this IServiceContainer self, Type iface, ITypeResolver implementation, Lifetime lifetime = Lifetime.Transient)
Parameters
Type Name Description
IServiceContainer self

The target IServiceContainer.
Type iface

The service interface to be registered. It can not be null and can be registered only once.
ITypeResolver implementation

The ITypeResolver is responsible for resolving the implementation. The resolved Type can not be null and must implement the iface interface. Additionally it must have only null or one constructor (that may request another dependecies). The resolver is called only once (on the first request) regardless the value of the lifetime parameter. For an implementation see the LazyTypeResolver<TInterface> class.
Lifetime lifetime

The Lifetime of the service.
Returns
Type Description
IServiceContainer

The container itself.
Remarks

You may register generic services (where the iface parameter is an open generic Type). In this case the resolver must return an open generic implementation.
