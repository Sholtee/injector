Factory(IServiceContainer, Type, Func<IInjector, Type, Object>, Lifetime)

Registers a new service factory with the given type. Factories are also services except that the instantiating process is delegated to the caller. Useful if a service has more than one constructor.

Declaration
public static IServiceContainer Factory(this IServiceContainer self, Type iface, Func<IInjector, Type, object> factory, Lifetime lifetime = Lifetime.Transient)
Parameters
Type Name Description
IServiceContainer self

The target IServiceContainer.
Type iface

The service interface to be registered. It can not be null and can be registered only once.
Func<IInjector, Type, Object> factory

The factory function that is responsible for the instantiation. Its call count depends on the value of the lifetime parameter. Note that the second parameter of the factory is never generic, even if you registered the factory for an open generic interface.
Lifetime lifetime

The Lifetime of the service.
Returns
Type Description
IServiceContainer

The container itself.
Remarks

You can register generic services (where the iface parameter is an open generic type).

Service(IServiceContainer, Type, String, Type, Lifetime)

Declaration
public static IServiceContainer Service(this IServiceContainer self, Type iface, string name, Type implementation, Lifetime lifetime = Lifetime.Transient)
Parameters
Type Name Description
IServiceContainer self

The target IServiceContainer.
Type iface

The service interface to be registered. It can not be null and can be registered only once (with the given name).
String name

The (optional) name of the service.
Type implementation

The service implementation to be registered. It can not be null and must implement the iface interface. Additionally it must have only null or one constructor (that may request another dependecies). In case of multiple constructors you can use the Factory(IServiceContainer, Type, Func<IInjector, Type, Object>, Lifetime) method or the ServiceActivatorAttribute.
Lifetime lifetime

The Lifetime of the service.
Returns
Type Description
IServiceContainer

The container itself.
Remarks

You may register generic services (where both the interface and the implementation are open generic types). The system will specialize the implementation if you request the concrete service.
